# OIBSIP - Python Programming Track

This repository contains the official project submissions developed during the Oasis Infobyte Python Programming Internship. All utilities are built using robust, modular architectures optimized to execute efficiently across standard Python 3 environments.

---

## 🛠️ Project Track Summary

### 📂 Task 2: Standard Body Mass Index (BMI) Analytics Engine
* **File Directory:** Swetha_Task2.py
* **Core Architecture:** Mathematical automation script containing rigorous floating-point numeric input validation blocks. 
* **Key Features:** Relies on robust defensive loops (`try-except`) to filter invalid non-numeric payloads. It processes dimensions in metric parameters ($kg/m^2$) and routes the finalized index score through conditional evaluation nodes to deliver immediate health status categorization.

### 📂 Task 3: Multi-Layer Cryptographic Passphrase Generator
* **File Directory:** Swetha_Task3.py
* **Core Architecture:** Leverages pseudo-random generation algorithms via internal core standard modules (`random` and `string`).
* **Key Features:** Implements a strict enforcement layer verifying that length variables remain above a minimum 8-character sequence. It uses boolean criteria mapping to build a customized, dynamic character array (uppercase, lowercase, numeric integers, and special punctuation). It guarantees at least one matching character for every selected rule set before executing a final randomized array shuffle.

### 📂 Task 5: Dual-Node Network Socket Chat Application
* **File Directory:** Swetha_Task5.py
* **Core Architecture:** Realized entirely through standard low-level network system interfaces (`socket` API framework) operating on localized network layers without third-party framework dependencies.
* **Key Features:** Houses both the network server router architecture and the client interface terminal inside a single operational matrix. It establishes full-duplex terminal communication on the loopback gateway (`127.0.0.1:9999`). The codebase includes connection exception overrides (`ConnectionRefusedError`) and data transmission control logic using automated serialization parameters (`utf-8`).

## 🚀 Execution & Verification Protocols

### 💻 Standard CLI Execution
1. Open a system terminal or command prompt window.
2. Navigate to the directory containing the source files and run:
   ```bash
   python Swetha_Task2.py
   ```
3. For **Task 5 (Chat Engine)**, execute the script in a primary terminal window and select option `1` to launch the listening server. Open a secondary terminal window, execute the script again, and select option `2` to initiate the connection handshake link.
